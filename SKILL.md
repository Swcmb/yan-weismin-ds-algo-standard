# 严蔚敏《数据结构》类 C 语言算法书写规范

本技能基于严蔚敏、吴伟民《数据结构》（第二版，清华大学出版社）教材附录中的「类 C 语言语法概要」和「算法书写规范」，AI 在编写数据结构算法代码时必须严格遵循。

## 一、适用范围

当出现以下任一情形时，必须遵循本规范：
- 编写数据结构基本操作的算法实现（顺序表、链表、栈、队列、串、树、图、查找、排序等）；
- 使用类 C 语言描述算法或书写伪代码；
- 用户明确要求按本规范或教材风格编写；
- 编写递归算法、链表操作、指针相关算法；
- 完成严蔚敏《数据结构》教材课后习题。

若用户明确要求使用标准 C/C++/Java/Python 等具体语言，以用户要求为准，本规范不强制套用。

## 二、预定义常量与类型

所有算法默认可使用以下预定义，无需重复声明：

```c
// 函数结果状态代码
#define  TRUE 1
#define  FALSE 0
#define  OK 1
#define  ERROR 0
#define  INFEASIBLE -1
#define  OVERFLOW -2

typedef  int  Status;   // Status 是函数的类型，其值是函数结果状态代码
typedef  int  bool;     // bool 是布尔类型，其值是 TRUE 或 FALSE
```

## 三、数据结构表示

- 数据结构的表示（存储结构）用类型定义 `typedef` 描述。
- 数据元素类型约定为 `ElemType`，由用户在使用时自行定义，不要在算法中固定为 `int` 或 `char`。

## 四、函数（算法）描述形式

基本操作的算法都用以下形式的函数描述：

```c
函数类型 函数名(函数参数表) {
    // 算法说明
    语句序列
} // 函数名
```

要点：
1. 函数的参数必须说明类型。
2. 算法中使用的辅助变量可以不作变量说明，必要时对其作用给出注释。
3. 变量命名约定：
   - `a, b, c, d, e` 等用作数据元素名；
   - `i, j, k, l, m, n` 等用作整型变量名；
   - `p, q, r` 等用作指针变量名。
4. 当函数返回值为函数结果状态代码时，函数类型定义为 `Status`。
5. 支持引用调用：在形参表中以 `&` 打头的参数即为引用参数（C++ 风格），用于将修改结果回传给调用者。

## 五、赋值语句

| 类别 | 形式 |
|---|---|
| 简单赋值 | `变量名 = 表达式;` |
| 串联赋值 | `变量名₁ = 变量名₂ = … = 变量名ₙ = 表达式;` |
| 成组赋值 | `(变量名₁, …, 变量名ₙ) = (表达式₁, …, 表达式ₙ);` |
| 结构赋值 | `结构名 = 结构名;` 或 `结构名 = (值₁, …, 值ₖ);` |
| 数组赋值 | `变量名[] = 表达式;` |
| 数组段赋值 | `变量名[起始下标..终止下标] = 变量名[起始下标..终止下标];` |
| 交换赋值 | `变量名₁ ↔ 变量名₂;` |
| 条件赋值 | `变量名 = 条件表达式 ? 表达式T : 表达式F;` |

## 六、选择语句

```c
// 条件语句 1
if (表达式) 语句;

// 条件语句 2
if (表达式) 语句;
else 语句;

// 开关语句 1（基于值）
switch (表达式) {
    case 值₁: 语句序列₁; break;
    …
    case 值ₙ: 语句序列ₙ; break;
    default: 语句序列ₙ₊₁;
}

// 开关语句 2（基于条件）
switch {
    case 条件₁: 语句序列₁; break;
    …
    case 条件ₙ: 语句序列ₙ; break;
    default: 语句序列ₙ₊₁;
}
```

## 七、循环语句

```c
// for 语句
for (赋初值表达式序列; 条件; 修改表达式序列) 语句;

// while 语句
while (条件) 语句;

// do-while 语句
do {
    语句序列;
} while (条件);
```

## 八、结束语句

| 类别 | 形式 |
|---|---|
| 函数结束语句 | `return 表达式;` 或 `return;` |
| case 结束语句 | `break;` |
| 异常结束语句 | `exit(异常代码);` |

## 九、输入输出语句

```c
scanf([格式串], 变量1, …, 变量n);     // 输入
printf([格式串], 表达式1, …, 表达式n); // 输出
```

通常可省略格式串。

## 十、注释

单行注释使用 `// 文字序列`。

## 十一、基本函数

| 函数 | 含义 |
|---|---|
| `max(表达式1, …, 表达式n)` | 求最大值 |
| `min(表达式1, …, 表达式n)` | 求最小值 |
| `abs(表达式)` | 求绝对值 |
| `floor(表达式)` | 求不足整数值（下取整） |
| `ceil(表达式)` | 求进位整数值（上取整） |
| `eof(文件变量)` 或 `eof` | 判定文件结束 |
| `eoln(文件变量)` 或 `eoln` | 判定行结束 |

## 十二、逻辑运算约定

- 与运算 `&&`：对于 `A && B`，当 `A` 的值为 0 时，不再对 `B` 求值（短路）。
- 或运算 `||`：对于 `A || B`，当 `A` 的值为非 0 时，不再对 `B` 求值（短路）。

## 十三、算法书写规范

编写算法时，必须遵循以下规范：

### 1. 算法说明（规格说明）

算法说明是完整算法不可缺少的部分，**必须在算法头（函数首部）之下以注释形式写明**，内容包括：
- 指明算法的功能；
- 参数表中各参数的含义和输入/输出属性；
- 引用的全局变量或外部变量及其作用、入口初值、应满足的限制条件（例如：链表是否带头结点、表中元素是否有序、按递增还是递减方式有序等）；
- 必要时陈述算法思想、采用的存储结构；
- **递归算法的说明特别重要**，应将其写成算法的严格定义。

要求：
- 算法说明应在开始设计算法时就写明，可在设计过程中补充修改，**切忌最后补写**；
- 书写（递归）算法规格说明时，应忽略它如何实现或假定它能实现；
- 算法说明书写得不好或不完全时，往往失去评判算法正确与否的标准。

### 2. 注释与断言

- 在难懂的语句和关键语句（段）之后加以注释，提高可读性；
- 注释要恰当，**并非越多越好**；注释句的抽象程度应略高于语句（段）本身，避免用 `// 增加 1` 注释 `i++;`；
- **断言**是注释的一种特殊写法，是陈述算法执行到某点时应满足条件的逻辑谓词，特别值得提倡；
- 最重要的断言是**算法入口断言**和 **else 分支断言**；
- 若算法不含参数合法性检测代码段，书写入口断言是最低限度要求。

### 3. 输入和输出

算法的输入输出可通过三种途径实现：
1. 通过 `scanf` 和 `printf` 语句（与计算环境外部交换信息）；
2. 以算法参数表中显式列出的参量作为输入/输出媒介（推荐）；
3. 通过全局变量或外部变量隐式传递（应尽量避免）。

若算法是定义在某个数据结构之上的几个操作之一，该数据结构可不列在参数表中。其他情况下应尽量避免使用第三种方法。

### 4. 错误处理

- 尽可能使用函数返回值返回算法执行状态（`OK` / `ERROR` / `INFEASIBLE` / `OVERFLOW` 等状态代码），便于调用者处理异常情况；
- 这有利于培养良好的程序设计习惯。

### 5. 语句选用和算法结构

- 赋值语句、`if` 分支语句、`while`（或 `for`）循环语句是最基本的三种语句，仅用此三种即可（且最好）应付一切算法设计；
- `switch` 语句是广义的 `if`，分支条件复杂时选用可避免 `if` 多重嵌套，鼓励使用；
- **一般情况下不准使用 `goto` 语句**，个别特殊情况除外。

应避免下列语句结构：

```c
// 应避免：循环嵌套不清晰
do {
    do {
        …
    } while …
    …
} while …

// 应避免：if 嵌套不明确
if ( … )
    if …
```

对于第二种情形，若难以改变，应对第二个 `if` 语句加上一对语句括号 `{ }`，明确条件成立时的作用范围。**语句的开/闭括号应对齐。**

### 6. 基本运算

- 若题目未明确要求用某种数据结构上的基本运算编写算法，**不得直接利用教科书中给出的基本运算**；
- 若非用不可，则要求将所用到的所有基本运算同时实现。

### 7. 几点建议

- 建议以图说明算法；
- 建议在算法书写完毕后，用边界条件的输入参数值验证算法能否正确执行（例如：对顺序表插入算法，空表是一个边界条件）。

## 十四、算法规格说明例释

以下为合格的算法规格说明示例，编写算法时应参照此风格：

**例 1**（第 2.10 题）
```c
Status DeleteK(SqList a, int i, int k)
// 本过程从顺序存储结构的线性表 a 中删除第 i 个元素起的 k 个元素。
```

**例 2**（第 2.39 题）
```c
float Evaluate(SqPoly pn, float x)
// 多项式 pn.data[i].coef 存放 aᵢ, pn.data[i].exp 存放 eᵢ(i=1, 2, …, m)。
// 本算法计算并返回 ∑(i=1到m) aᵢxᵉⁱ。不判别溢出。
// 此外，入口时要求 0≤e₁<e₂<…<eₘ，算法内不对此再作验证。
```

**例 3**（第 3.15 题的入栈 push 算法）
```c
void push(TwoWayStack &tws, int i, ElemType x, bool &overflow)
// 两栈(标号 0, 1)共享空间 tws.elem[0..m-1]，栈底在两端。
// tws.top[0] 和 tws.top[1] 为两栈顶指针。
// tws.data[0..tws.top[0]] 为栈 0，
// tws.data[tws.top[1]..m-1] 为栈 1。本算法将 x 推入栈 i(i=0,1)。
// 若入口时栈满，则不改变栈且返回 overflow 为 TRUE。
```

**例 4**（第 3.30 题的出列算法）
```c
bool dequeue(ElemType &x)
// 循环队列 squeue[(m+rear－quelen+1) MOD m .. rear] 之上的元素出列操作。
// quelen 为队列长度。当 quelen=0 时，返回 FALSE；
// 否则返回 TRUE，且用 x 返回出列的元素值。
```

## 十五、完整示例

下面给出一个符合本规范的完整算法示例，供参照：

```c
Status ListInsert_Sq(SqList &L, int i, ElemType e)
// 在顺序线性表 L 中第 i 个位置之前插入新的元素 e，
// i 的合法值为 1 ≤ i ≤ ListLength_Sq(L)+1。
// 入口断言：1 ≤ i ≤ L.length+1；L 为带头容量的顺序表。
// 返回值：成功返回 OK；i 非法返回 ERROR；溢出返回 OVERFLOW。
{
    if (i < 1 || i > L.length + 1) return ERROR;   // i 值非法
    if (L.length >= L.listsize) {                  // 当前存储空间已满，扩充
        ElemType *newbase = (ElemType *)realloc(L.elem,
                              (L.listsize + LISTINCREMENT) * sizeof(ElemType));
        if (!newbase) return OVERFLOW;             // 存储分配失败
        L.elem = newbase;
        L.listsize += LISTINCREMENT;
    }
    ElemType *q = &(L.elem[i - 1]);                // q 为插入位置
    for (ElemType *p = &(L.elem[L.length - 1]); p >= q; --p)
        *(p + 1) = *p;                             // 插入位置及之后的元素右移
    *q = e;                                        // 插入 e
    ++L.length;                                    // 表长增 1
    return OK;
} // ListInsert_Sq
```

## 十六、自检清单

编写完成后，请对照以下清单自检：

- [ ] 算法头之下是否有算法说明（功能、参数含义、输入输出属性、限制条件）？
- [ ] 数据元素类型是否使用 `ElemType` 而非具体类型？
- [ ] 返回状态代码的函数是否使用 `Status` 类型？
- [ ] 需要回传的参数是否使用了引用参数 `&`？
- [ ] 变量命名是否遵循约定（a-e 元素、i-n 整型、p-r 指针）？
- [ ] 是否避免了 `goto` 语句？
- [ ] 关键语句和难懂处是否有注释？是否书写了入口断言？
- [ ] 是否用边界条件（如空表、满栈、单元素）验证过算法？
- [ ] 若引用了基本运算，是否同时实现了这些基本运算？
